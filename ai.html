§04 The fall-kit substrate · what every seed gets

Every seed in the estate includes the fall-kit library (prime 1301), which provides:

  • T0 cascade · template-based responses · deterministic · zero-cost · always works (no network)
  • T2 cascade · WebLLM in-browser inference · Llama-1B/3B, Qwen-7B, Llama-8B/70B · runs on user's GPU
  • T3 cascade · BYOK frontier models · Anthropic, OpenAI, Google · user-supplied API key
  • WebRTC mesh · peer-to-peer coordination via fall-signal BroadcastChannel
  • Sovereign signing · Ed25519 keypairs via Web Crypto API
  • Mint hook · one-click fork minting via kcc-mint
  • Help tab · standardised onboarding for each seed

The cascade tiers fall through automatically based on availability. T0 always succeeds. T2 requires WebGPU. T3 requires API key.

§11 Sovereignty contract · what we guarantee

We promise: every seed runs entirely client-side. No data leaves your browser unless you opt in (via BYOK API key or kp2p mesh). No telemetry, no analytics, no tracking. MIT license forever. If the operator disappears, the tools keep working.

We do not promise: uptime guarantees, 24/7 support, enterprise SLAs, formal compliance certifications. These can be added per-tenant via the L1 compliance seed but are not pre-bundled.

What you own: the HTML file. Once downloaded, it is yours. No update can remove your access. No vendor can take it back.
